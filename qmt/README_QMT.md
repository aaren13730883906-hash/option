# QMT 期权回测接入步骤

当前目标是先验证 QMT 是否能在历史回测日期访问已经到期的
`588000.SH`、`159915.SZ` 期权。这个能力是复现 v1.2（320.23%）
策略的硬前提。

## 第一步：运行环境探针

1. 将 `qmt_option_history_probe.py` 复制到 QMT 策略目录。
2. 回测周期选择 `1m`。
3. 回测日期选择 `2025-07-23` 全天。
4. 不需要填写交易账号，脚本不会调用 `passorder`。
5. 运行后复制所有以 `[QMT_PROBE]` 开头的日志。

首条日志必须是：

`[QMT_PROBE] FIRST_BAR 20250723093000`

如果出现 `FATAL_WRONG_BACKTEST_DATE`，说明 QMT 实际使用的回测日期不是
2025-07-23；探针会停止检测，以免把“合约在当日已经到期”误判为“没有历史
行情”。

探针会验证：

- `588000.SH` 和 `159915.SZ` 是否能返回期权合约列表；
- 合约列表在 2025-07-23 是否包含有效历史合约；
- 深圳期权代码后缀究竟是 `.SZO`、`.SZ` 还是其他形式；
- 两张已知历史期权能否返回变化中的 1 分钟价格；
- `open/high/low/close/volume/amount` 的实际返回结构；
- `optType`、`OpenDate`、`ExpireDate`、行权价和合约乘数的真实字段格式。

## 为什么必须先跑探针

交接文档只验证了 `510300.SH` 和当前可见期权。当前策略需要：

- 科创 ETF：`588000.SH`；
- 创业板 ETF：`159915.SZ`；
- 历史已到期期权；
- 1 分钟 ETF/期权 OHLCV；
- 动态 DTE 10-35 合约池。

如果 QMT 的 `get_option_undl_data` 只返回当前挂牌合约，动态策略无法直接
回测 2025 年历史期权。此时需要改用“外部历史合约表 + QMT 直接读取历史
期权代码”的架构。

## 第二步需要提供的信息

请把以下内容发回：

1. 探针完整日志；
2. QMT版本号和券商名称；
3. 回测界面可选的最细周期（确认是否为1分钟）；
4. 回测手续费设置页面截图或具体数值；
5. 若要让策略调用QMT撮合而非内部影子账户，提供官方期权回测
   `passorder` 示例中的 `opType/orderType/prType` 说明；
6. QMT策略目录的绝对路径，以及策略是否能读取同目录CSV文件。

收到探针结果后，主策略将按实际返回结构实现，不猜测接口。

## 纯期权分钟数据探针

如果ETF的1分钟历史数据不足，使用 `qmt_option_only_probe.py` 单独验证历史
期权数据：

1. 回测周期选择 `1m`；
2. 开始和结束日期都选择 `2025-07-23`；
3. 运行后复制所有 `[QMT_OPTION_ONLY]` 日志。

该脚本直接将 `10009568.SHO`、`90005721.SZO` 设置为股票池，不依赖
`588000.SH` 或 `159915.SZ` 产生回测事件，也不会发送订单。

## 2025-11-10至2026-06-30合约可用性探针

使用 `qmt_option_range_availability_probe.py` 检查QMT当前合约目录在历史
日期是否能提供符合DTE 10-35的期权及分钟行情：

1. 回测周期选择 `1m`；
2. 开始日期选择 `2025-11-10`；
3. 结束日期选择 `2026-06-30`；
4. 复制全部 `[QMT_RANGE]` 日志。

每日结果中：

- `directory` 是QMT返回的合约目录数量；
- `active` 是挂牌日和到期日覆盖该历史日期的合约数量；
- `dte_10_35` 是符合策略DTE规则的数量；
- `priced` 是当日首根分钟K线可以读取正价格的合约数量。

## 历史期权专用接口探针

`get_option_undl_data` 只适合当前合约池。使用
`qmt_historical_option_api_probe.py` 验证历史期权专用接口：

- `ContextInfo.get_option_list`；
- `ContextInfo.get_his_contract_list`。

该探针可在任意能够启动的1分钟回测日期运行。若历史列表为空，需要先在
QMT数据管理中手动补充“过期合约列表”，然后重新运行。复制全部
`[QMT_HIST_API]` 日志。

## v1.2 双标实时影子模拟

`qmt_v12_dual_live_paper.py` 用QMT实时1分钟行情运行内部10万元模拟账户。
脚本不会调用 `passorder`，不需要填写账号，也不会产生真实委托。

### 当前存续合约回放验收

在迁移到QMT模拟交易前，先利用当前仍存续期权的历史分钟K线完成两轮验收。
推荐回测区间为 `2026-05-15` 至 `2026-06-30`，其中5月15日至6月16日仅
用于ETF日线预热，策略和期权池从 `PAPER_START_DATE = 20260617` 开始。

第一轮为执行冒烟测试：

1. 保持 `EXECUTION_SMOKE_TEST = True`；
2. 保持 `SMOKE_TEST_DATE = 20260617`、时间 `0946`；
3. QMT回测周期选 `1m`；
4. 运行后检查 `POOL_READY`、`SMOKE_ORDER_OPENED`、`BUY`、`SELL` 和
   `TRADE_CLOSED`；
5. 该测试单只验证选约、反算IV、仓位、内部成交和退出链路，不计入策略
   收益评价。

第二轮为真实策略回放：

1. 改为 `EXECUTION_SMOKE_TEST = False`；
2. `FORCE_TEST_DIRECTION` 保持空字符串；
3. 使用相同回测区间再次运行；
4. 检查 `DAILY_STATE`、`OPENING_SIGNAL`、`FALLBACK_SIGNAL`、
   `OPTION_SELECTED` 以及完整开平仓日志；
5. `SELECT_NO_MATCH` 会列出因无分钟K线、IV、Delta、趋势或成交量被过滤的
   合约数量。

回测历史时逐合约IV优先用当时的期权成交价反算，日志显示
`iv_source=reversed`。这样不会误把当前IV当成历史IV。该回放只能验证当前
仍存续合约覆盖到的短时间窗口，不能替代320.23%完整历史收益回测。

运行步骤：

1. 将完整脚本复制到QMT新建的Python模型；
2. 周期选择 `1m`；
3. 将脚本顶部 `PAPER_START_DATE` 改为实际开始测试日；
4. 确保数据管理中已补充 `588000.SH`、`159915.SZ` 的日线、1分钟数据；
5. 使用QMT“运行”模式启动，观察 `[V12_PAPER]` 日志。

启动检查：

- `DAILY_PRELOAD_OK ... days=` 应至少为21；
- `DAILY_STATE ... ready=True` 表示日线方向过滤器可用；
- `POOL_READY` 表示当日DTE 10-35期权池已建立；
- `BUY`、`ADD`、`SELL`、`TRADE_CLOSED` 是内部模拟成交。

`FORCE_TEST_DIRECTION` 默认必须留空。只有在测试行情和选约接口、且日线预热
失败时，才临时填写 `"call"` 或 `"put"`；该模式不代表正式策略信号。

实时版保留双标择优、科创备用、创业无备用、DTE 10-35、实时逐合约IV与
Delta、仓位和退出规则。相对历史正式回测有三项明确差异：

- 为避免实时先后信号造成双边同时开仓，早盘双标统一在09:43比较后成交；
- 使用实时逐合约IV，不使用历史日级IV Rank前视数据；
- QMT不提供上一日逐合约成交量时，选约流动性评分使用当日累计分钟成交量。

停止或重新运行模型会重置内部模拟资金和持仓；QMT原生成交列表保持为空。
